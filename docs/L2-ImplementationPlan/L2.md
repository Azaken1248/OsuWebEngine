## TDD Divergences Found

The TDD's parser pseudocode is a reasonable sketch but **wrong in nine places**.
Implement what lazer does, not what the TDD says.

### `.osr`

| # | TDD ¬ß2 says | lazer actually does | Source |
|---|---|---|---|
| D1 | Score ID is `i64`, present for "‚â• 2018 format" | `version >= 20140721` ‚Üí `i64`; `version >= 20121008` ‚Üí **`i32`**; else **absent** | `LegacyScoreDecoder.cs` L107-110 |
| D2 | ‚Äî (not mentioned) | `version >= 30000001` ‚Üí an extra compressed score-info byte array follows | L117-118 |
| D3 | Frame delta parsed as `f64` | Parsed as **`int`**, falling back to `round(float)` ‚Äî stable forbids fractional deltas to avoid FP accumulation | L304-306 |
| D4 | Terminator: `if delta == -12345 { break }` | **String** compare `split[0] == "-12345"`, and **`continue`**, not `break`. Its delta is never accumulated | L282-286 |
| D5 | `absolute_time` starts at `0` | Starts at `beatmapOffset` = 24 ms if the beatmap is format < 5, else 0 | L98, L270 |
| D6 | ‚Äî | Coordinates bounded by `MAX_COORDINATE_VALUE = 131072`; out-of-range is an error | `Parsing.cs` L14 |
| D7 | ‚Äî | **Four stable replay quirks** applied post-parse (see below) | L319-351 |

**D7 ‚Äî the stable `ReplayWatcher` quirks**, in order:

1. If `frames.len() >= 2 && frames[1].time < frames[0].time`:
   `frames[1].time = frames[0].time; frames[0].time = 0`
2. If `frames.len() >= 3 && frames[0].time > frames[2].time`:
   `frames[0].time = frames[1].time = frames[2].time`
3. Stable writes two sentinel frames at the start, both at position
   **(256, ‚àí500)**. Drop index 1 if it is a sentinel, then index 0.
4. Never allow backwards time traversal: skip any frame whose time is less
   than the previously emitted frame's.

None of these are in the TDD. Without them, frame timing is wrong at the start
of essentially every stable replay.

> The collinear-`P` ‚Üí `LINEAR` downgrade is a **parser** rule and does not

> because the parser rewrites it to `LINEAR` first. The flattener fallback
> still governs lazer-format maps and any case that slips through. See
> [ADR-021](../ADR_Registry.md).

## Design Decisions

### 1. `ByteReader` over `std::io::Read`
The TDD sketches `impl Read`. We parse from a `&[u8]` that is already fully in
memory (the WASM boundary hands us a `Uint8Array`), so a cursor over a slice is
simpler, allocation-free, and cannot block. `parser/binary.rs`.

### 2. Integer frame deltas (D3)
Deltas are accumulated as `i64` and only widened to `f64` when stored on the
frame. This reproduces stable's integral-delta guarantee exactly and sidesteps
the accumulation error the lazer source comments call out.

### 3. Bounded LZMA, streaming
`lzma_rs::lzma_decompress` into a `Vec` guarded by a 256 MB cap
(BRD ¬ß14.1). The cap is checked against the *declared* size before allocating,
and enforced again on the actual output.

### 4. Unknown curve type ‚Üí Catmull, not an error (D8)
Faithful to lazer. A parser that rejects what the game accepts would fail on
real maps.

### 5. Offsets applied at parse time
`EARLY_VERSION_TIMING_OFFSET` is baked into hit object and timing point times
during parsing, exactly as `LegacyBeatmapDecoder` does, so downstream layers
never need to know the beatmap's format version.

### 6. `slides` retained, not `repeatCount`
`SliderData::repeat_count` keeps the file's `slides` semantics (1 = no repeat),
matching the existing L3 model docs. lazer's internal `repeatCount` is
`slides - 1`; the conversion is a subtraction at the point of use, and doing it
here would silently change the model's documented contract.

### Gaps found during exit review

Two L2 tasks were silently under-delivered and are now closed:

- **L2-15** called for 15 edge-case fixtures; 7 existed. Now 18.
- **`tests/fixtures/malformed/` did not exist**, so none of SEC-001..007 had ever
  been written ‚Äî leaving the engine's entire untrusted-input surface untested by
  anything except the fuzzer. The corpus and the seven tests now exist.
  `lzma_bomb.osr` is a genuine bomb (44 KB expanding to 300 MB, 7074:1),
  generated by compressing 300 MB of zeroes rather than forging a size header, so
  it exercises the streaming decompression path instead of an early sanity check.
