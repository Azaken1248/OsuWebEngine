# L1: Core Math — Implementation Plan (v2)

## Goal

Implement all curve primitives (Bézier, Catmull-Rom, Perfect Arc, Linear) and the composite `SliderPath` with arc-length parameterization. All positions must be within **0.5 osu!px** of osu!lazer's `SliderPath` output.

> [!IMPORTANT]
> **Guiding principle**: We are reproducing osu!lazer behavior, not inventing algorithms. danser-go is our **reading aid** (easier to follow than C#), but osu!lazer is the **specification**. If they ever diverge, lazer wins (BRD §7.1).

## Design Decisions

### 1. f64 throughout
TDD §11.1 mandates f64 to match C# `double`. Conversion to f32 happens only at the WASM boundary (L7).

### 2. Flattening: iterative, not recursive
Bézier flattening uses an explicit stack (`Vec<Vec<Vec2>>`) to emulate recursion, preventing stack overflow on pathological high-degree curves. Mirrors danser's `toFlatten` stack approach.

### 3. Circular arc fallback → **Bézier**, not linear
~~Follow lazer's actual behavior: collinear → linear fallback. No explicit radius > 500 check (danser doesn't have one).~~

**Corrected 2026-07-14.** All three arc fallbacks route to the **Bézier**
approximation. In `SliderPath.calculateSubPath` each condition `break`s out of
the `PerfectCurve` case, and the switch falls through to
`BSplineToPiecewiseLinear`:

| Condition | Source |
|---|---|
| Control point count != 3 | `SliderPath.cs` L345 |
| Collinear / invalid arc | `SliderPath.cs` L351 |
| Arc needs >= 1000 points | `SliderPath.cs` L359 |

The 1000-point cap matters beyond fidelity: the point-count formula diverges as
`r → ∞`, so without it a crafted `.osu` can force an unbounded allocation. The
"no radius > 500 check" half of the original decision was right; the "fall back
to linear" half was not.

### 4. Catmull endpoint extrapolation
Missing endpoints use `v4 = v3 + v3 - v2` (equivalently `p2 + (p2 - p1)`); the
missing *start* duplicates (`v1 = v2`). Matches `PathApproximator.cs` L156-159.

### 5. Catmull emits **two points per step**
`CatmullToPiecewiseLinear` pushes both `t = c/detail` and `t = (c+1)/detail` on
every step, yielding `detail * 2` points per segment with duplicated interior
vertices (`PathApproximator.cs` L161-164). This looks redundant — it creates
zero-length segments — but `OptimiseCatmull` keys its knot detection off
`(i + 1) % (catmull_detail * 2) == 0`, so collapsing it breaks that pass.

### 6. Catmull optimisation is mandatory
`Slider.cs` L45 constructs its path as `new SliderPath { OptimiseCatmull = true }`
— it is **hardcoded on** for every osu! Standard slider, not opt-in. The pass
strips vertices closer than 6px and returns the length it removed, which seeds
the length accumulator (`SliderPath.cs` L375-419, L428). Skipping it inflates the
length of any Catmull slider with duplicate knots.

### 7. SliderPath resolves length by shortening **or extending**
Path is trimmed when longer than `pixel_length`, and **extended** along the final
segment's direction when shorter (`SliderPath.cs` L482). Extension is suppressed
when the last two path points coincide, since there is no direction to extend
along (L450). An earlier draft described only the trimming half.

### 6. No micro-optimization
No SIMD, SmallVec, unsafe, or custom allocators. Correctness first. Optimization happens after differential tests pass.

### 7. Curve trait abstraction
Common `Curve` trait for all curve types. Enables future extensibility (editor tooling, mania) without coupling consumers to concrete types.

### 8. Centralized constants
All epsilon/tolerance values live in `math/constants.rs`. No magic numbers scattered across files. Easy to audit and tune.

/// Squared length — avoids sqrt for flatness checks.
#[inline]
pub fn length_sq(self) -> f64 {
    self.x * self.x + self.y * self.y
}

/// Approximate equality within epsilon (for tests and comparisons).
#[inline]
pub fn approx_eq(self, other: Self, epsilon: f64) -> bool {
    (self.x - other.x).abs() < epsilon && (self.y - other.y).abs() < epsilon
}

#### [NEW] bezier.rs

Bézier curve flattening. Behavior validated against lazer `PathApproximator`, verified readable via danser `bezierapproximator.go`.

```rust
/// Checks if the 2nd-order derivative of the control polygon is
/// within tolerance (i.e., the curve is locally "flat enough").
///
/// Ref: danser IsFlatEnough() — checks curvature via finite differences.
fn is_flat_enough(points: &[Vec2]) -> bool

/// Subdivides a Bézier at t=0.5 into left and right halves.
/// Uses De Casteljau's algorithm (in-place midpoint averaging).
///
/// Ref: danser Subdivide()
fn subdivide(points: &[Vec2], left: &mut [Vec2], right: &mut [Vec2])

/// Approximates a single Bézier using the subdivision midpoints.
///
/// Ref: danser Approximate()
fn approximate_segment(points: &[Vec2], output: &mut Vec<Vec2>)

/// Adaptively flattens a Bézier curve to a polyline.
///
/// Uses an **iterative** stack (not recursion) to avoid overflow
/// on pathological curves. Reserves estimated capacity.
///
/// Ref: danser CreateBezier() — iterative DFS with toFlatten stack.
pub fn flatten_bezier(points: &[Vec2]) -> Vec<Vec2>

/// Splits control points at repeated-point boundaries into segments.
///
/// `[A, B, C, C, D, E]` → `[[A, B, C], [C, D, E]]`
///
/// Ref: danser processBezier()
pub fn split_and_flatten_bezier(points: &[Vec2]) -> Vec<Vec2>
```

**Allocation note**: `flatten_bezier()` calls `Vec::with_capacity(points.len() * 2)` as initial estimate.

#### [NEW] catmull.rs

Catmull-Rom spline evaluation. Behavior validated against lazer, verified via danser `catmull.go`.

```rust
/// Evaluates a single Catmull-Rom segment at parameter t ∈ [0, 1].
///
/// Formula: q(t) = 0.5 × ((2P₁) + (-P₀+P₂)t + (2P₀-5P₁+4P₂-P₃)t² + (-P₀+3P₁-3P₂+P₃)t³)
///
/// Ref: danser findPoint()
fn catmull_point(p0: Vec2, p1: Vec2, p2: Vec2, p3: Vec2, t: f64) -> Vec2

/// Flattens a Catmull-Rom spline across all segment pairs.
///
/// - Missing start: p0 = p1 (duplicate first point)
/// - Missing end:   p3 = p2 + (p2 - p1) (extrapolate)
/// - Generates CATMULL_DETAIL samples per segment.
///
/// Ref: danser processCatmull() + ApproximateCatmullRom()
pub fn flatten_catmull(points: &[Vec2]) -> Vec<Vec2>
```

**Allocation note**: `Vec::with_capacity((points.len() - 1) * CATMULL_DETAIL)`.

#### [NEW] circular_arc.rs

Perfect circular arc. Behavior validated against lazer, verified via danser `cirarc.go` + `approximation.go`.

```rust
/// A circle defined by 3 points, with arc traversal parameters.
struct CircularArc {
    centre: Vec2,
    radius: f64,
    start_angle: f64,
    total_angle: f64,
    direction: f64,  // +1.0 or -1.0
}

impl CircularArc {
    /// Computes circumcenter, radius, and arc direction from 3 points.
    /// Returns None if points are collinear (|D| < COLLINEAR_EPSILON).
    ///
    /// Ref: danser NewCirArc()
    fn new(a: Vec2, b: Vec2, c: Vec2) -> Option<Self>

    /// Position at parameter t ∈ [0, 1].
    ///
    /// Ref: danser PointAtL() — cos/sin on interpolated angle.
    fn point_at(&self, t: f64) -> Vec2
}

/// Checks if 3 points are collinear via cross product.
pub fn is_straight_line(a: Vec2, b: Vec2, c: Vec2) -> bool

/// Flattens a perfect arc to a polyline with adaptive segment count.
///
/// Segment count = max(2, ceil(totalAngle / (2 * acos(1 - ARC_TOLERANCE / radius))))
///
/// Fallbacks:
///   - Collinear points → linear interpolation
///   - > 3 control points → Bézier fallback
///   - < 3 control points → linear
///
/// Ref: danser ApproximateCircularArcLazer() + processPerfect()
pub fn flatten_arc(points: &[Vec2]) -> Vec<Vec2>
```

#### [NEW] slider_path.rs

Composite slider path with arc-length parameterized lookup. This is the core public API.

```rust
/// A fully resolved slider path with arc-length parameterization.
///
/// After construction, `position_at(t)` returns the position at
/// arc-length fraction t ∈ [0, 1] in O(log N) time.
pub struct SliderPath {
    /// Original curve type from the beatmap.
    curve_type: CurveType,
    /// Original control points from the beatmap.
    control_points: Vec<Vec2>,
    /// Desired pixel length (may be shorter than full curve).
    pixel_length: f64,
    /// Flattened polyline points (post-clamping).
    path_points: Vec<Vec2>,
    /// Cumulative arc lengths: cumulative_lengths[i] = distance from
    /// path_points[0] to path_points[i].
    cumulative_lengths: Vec<f64>,
}

impl SliderPath {
    /// Constructs a SliderPath from control points and desired length.
    ///
    /// 1. Dispatches to flatten_bezier/flatten_catmull/flatten_arc/linear
    /// 2. Builds cumulative length table
    /// 3. Clamps to pixel_length (trims + interpolates last point)
    ///
    /// Ref: danser NewMultiCurveT()
    pub fn new(
        curve_type: CurveType,
        control_points: Vec<Vec2>,
        pixel_length: f64,
    ) -> Self

    /// Position at arc-length fraction t ∈ [0, 1].
    ///
    /// Uses `slice::partition_point()` for O(log N) binary search.
    /// **Zero heap allocations** after construction.
    ///
    /// Ref: danser PointAtLazer()
    pub fn position_at(&self, t: f64) -> Vec2

    /// Total arc length after clamping.
    pub fn length(&self) -> f64

    /// Pre-computes n evenly-spaced points for rendering.
    pub fn render_points(&self, n: usize) -> Vec<Vec2>

    // Accessors
    pub fn curve_type(&self) -> CurveType
    pub fn control_points(&self) -> &[Vec2]
    pub fn pixel_length(&self) -> f64
    pub fn path_points(&self) -> &[Vec2]
}
```

**Binary search**: Uses `slice::partition_point()` (Rust stdlib, no custom implementation).

**Clamping**: Follows danser's `NewMultiCurveT()` exactly — trim cumulative lengths exceeding `pixel_length`, interpolate the last point along the final segment direction.

### Benchmark Baselines

performance baseline for future optimization (L8+).
